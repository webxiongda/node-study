# Day 01 — 项目任务：开发环境诊断工具

## 业务背景

你入职了一家公司，团队有多个开发者，经常出现"我本地跑得好好的，别人跑不起来"的问题。
原因通常是：Node.js 版本不同、缺少环境变量、操作系统差异。

你的任务是写一个 **`dev-doctor.js`** 诊断脚本，帮助团队快速排查环境问题。

## 技术要求

使用 `process` 对象和 `os` 模块，**不允许安装任何第三方包**。

## 输入输出

**运行方式：**
```bash
node dev-doctor.js
node dev-doctor.js --min-node=18  # 指定最低 Node.js 版本要求
```

**期望输出格式：**
```
╔══════════════════════════════════════════╗
║         开发环境诊断报告                  ║
╚══════════════════════════════════════════╝

[✅ 通过] Node.js 版本: v20.11.0 (要求 >= 18)
[✅ 通过] 操作系统: macOS 14.2.1 (darwin/arm64)
[✅ 通过] 内存: 总量 16.0 GB，可用 4.2 GB (26.3% 已用)
[⚠️ 警告] NODE_ENV 未设置（建议设置为 development）
[✅ 通过] 工作目录可写: /Users/alice/projects/myapp

==== 诊断结果 ====
通过: 4 项  警告: 1 项  失败: 0 项
```

## 验收标准

- [ ] 正确显示 Node.js 版本，并与 `--min-node` 参数比较（版本低于要求时显示 `[❌ 失败]`）
- [ ] 显示操作系统类型、版本、架构
- [ ] 显示内存总量和可用量（单位 GB，保留1位小数）
- [ ] 检查 `NODE_ENV` 是否已设置
- [ ] 统计通过/警告/失败数量并汇总
- [ ] 退出码：全部通过为 `0`，有失败项为 `1`

## 常见坑

1. **版本比较不能用字符串比较**：`'v20' > 'v18'` 是字符串比较，在版本号位数不同时会出错。用 `parseInt` 提取主版本号比较。
2. **内存单位换算**：`os.totalmem()` 返回字节数，换算 GB 要除以 `1024³`（即 `1073741824`），不是 `1000³`。
3. **`process.exit(code)` 要在所有输出完成后调用**，否则最后一行可能打印不出来（stdout 是异步的）。
