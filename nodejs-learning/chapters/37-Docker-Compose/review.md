## 高频面试题（含答案）

**Q1：Docker-Compose为什么值得在项目里做？**
A：它体现的是工程化能力，不只是功能实现。回答时要结合业务场景、技术方案、失败处理和验证方式。

**Q2：你如何判断方案是否有效？**
A：看是否有可复现的输入输出、异常路径验证、日志或指标，以及是否能在 README 中写清楚操作步骤。

**Q3：本章方案有什么替代方案？**
A：替代方案要从复杂度、成本、团队维护和项目阶段比较。面试时不要说唯一答案，要说当前阶段为什么选这个。

**Q4：线上出现问题怎么排查？**
A：按入口、业务逻辑、外部依赖、配置环境、日志指标的顺序排查，先定位范围再改代码。

**Q5：如何把本章内容写成项目亮点？**
A：用“业务问题 + 技术方案 + 结果指标 + 取舍”表达，而不是罗列技术名词。
