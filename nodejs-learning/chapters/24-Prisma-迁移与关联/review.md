# Day 24 — Prisma 迁移与关联：复习文档

## 核心知识点总结

| 概念 | 要点 |
|------|------|
| 隐式多对多 | schema 里互相引用，Prisma 自动建中间表，适合简单场景 |
| 显式多对多 | 手动建中间表 Model，适合有额外字段的场景 |
| onDelete | Cascade（级联删）/ SetNull（置空）/ Restrict（拒绝删除父记录） |
| 批处理事务 | `$transaction([q1, q2])`：并行，适合分页 count+findMany |
| 交互式事务 | `$transaction(async tx => ...)`：串行，抛错自动回滚 |
| 嵌套写入 | `create/connect/connectOrCreate/set` 在关联字段里操作 |

## 高频面试题

**Q1：Prisma 如何实现数据库事务？**
两种方式：(1) 批处理 `$transaction([...queries])` 并行执行多个操作；(2) 交互式 `$transaction(async tx => {...})` 串行执行，在回调里用 `tx` 代替 `prisma` 保证操作在同一事务内，任何操作抛错则全部回滚。

**Q2：如何安全地在生产环境迁移数据库？**
使用 `prisma migrate deploy`（不是 dev）。步骤：(1) 新字段先加可空，(2) 部署代码，(3) 后台任务填充数据，(4) 再加约束。关键是向后兼容，不要一步到位。

**Q3：隐式多对多和显式多对多什么时候用哪个？**
需要在关联本身存额外信息（关联时间、权重、状态）时用显式；纯粹的关联关系（只需要知道谁关联谁）用隐式，更简洁。
